blau-drill

A single-operator, pure-browser instrument (Gleam → JavaScript via Lustre, no backend, no install) that loads a KiCad board, drives the modified Two Trees Bluer over the Web Serial API, fits an affine board → machine transform from human-located fiducials, and streams the drilling — dry-run first.

00Goals, principles & invariants

The foundation

Goals

  • Take a KiCad board through one drilling session, end to end, in one app: load → register → align → dry-run → drill.
  • Replace the manual chain (re-export, mirror flag, fiducial G92, pcb2gcode, postprocess_drill.py) with a single fitted affine transform that absorbs offset, rotation, mirror, and skew.
  • Give a human a visual, jog-driven alignment loop with live position feedback and trustworthy residuals.
  • Generate Marlin-correct G-code natively (spindle on shared laser PWM as M3 S255).

Non-goals load-bearing

  • Not a multi-user / multi-machine service. One bench, one printer, one operator.
  • Not a persistent job database — each session is a fresh upload (ephemeral state).
  • Not a CAM suite. No trace isolation, no laser-ablation pass — drilling only (for now).
  • Not a replacement for OctoPrint as a general print host — it owns the serial port only for the duration of a drilling session.

Runtime constraints come with the architecture

  • No backend, no install — the win: the whole app is Gleam compiled to JavaScript (Lustre), served as static files. The operator just opens a URL; there is no server to run, no server-side rendering, no database. All state lives in the Lustre model in the browser.
  • Chromium-only: it talks to the printer directly over the Web Serial API (navigator.serial), available in Chrome / Edge / Opera 89+. Firefox and Safari are not supported.
  • Secure context required: Web Serial only works over HTTPS, localhost, or file://.
  • User-gesture connect: the port chooser can only open from an explicit operator click — the app can never silently grab the port.

Core principles

P1
The board is a value; the printer is the only identity. Parsed boards, transforms, and G-code are immutable values that flow through pure functions. Exactly one stateful reference exists: the serial PrinterConnection.
P2
Alignment is a solved transform, not a procedure. Registration captures correspondences; the transform is fitted and carries its own residuals. No G92 ritual in the domain — that's an implementation detail of streaming.
P3
Every boundary parses into a precise type. The .drl, Gerber, and each Marlin serial line are parsed at the edge into domain values; malformed input fails loudly there, never deep inside.
P4
Illegal sequencing is unrepresentable. You cannot stream a drill without an Alignment; you cannot fit an Alignment without ≥3 non-collinear points. The Job state machine makes the only legal path the only typed path.
P5
The dangerous defaults are enforced, not documented. Energize-before-jog, residual gate before drilling, dry-run before real — each is a state the machine must pass through, not a README sentence.

Invariants — each paired with the representation that enforces it

Invariant (must always hold)Made unrepresentable by…
An Alignment exists only when solvable. No public constructor. Only alignment.fit(corrs) returns Ok(Alignment(..)), and only for ≥3 non-collinear points; otherwise Error(Degenerate). Fewer points are a PendingAlignment — a different Gleam type.
Drilling cannot start without a valid alignment. gcode_program.build takes an Alignment as a required argument — there is no overload that accepts raw holes. The "Drill" event only exists in Job state Aligned.
Residuals over tolerance block the real run. Job transition Aligned → Drilling is guarded by residuals.max ≤ tol; a failing fit lands in AlignmentRejected, which has no drill event.
You never jog a de-energized axis (the 1–2 mm snap). PrinterConnection state Idle matches no Jog command — the pure core returns a NotEnergized refusal and writes nothing. The only path to Jogging runs an energize step (M17) first.
Real drilling is never the first thing that runs. Job has no Aligned → Drilling case clause — it routes through DryRun, whose completion is the precondition for the real-run event.
A hole's machine coordinate is always the transform of its board coordinate. Holes are stored only in board space. Machine coordinates are computed on demand by transform2d.apply — there is no settable machine-coordinate field to drift out of sync.

The central type lattice

The registration → alignment progression is the spine. Illegal states (a "drilling job with no transform", a "transform from 2 points") have no node — they cannot be constructed.

02Lens-driven design rationale

Why it's shaped this way

DEPTH

Two deep modules carry the hard parts

PrinterConnection hides the entire Marlin serial protocol — port lifecycle, line numbering, checksums, the ok/resend handshake, M114 polling, flow control — behind a small set of verbs: Energize/Release, Jog, MoveTo, Where, Stream, Halt, Reconnect. Alignment hides a least-squares affine solve behind one function: alignment.fit(correspondences). A caller needs to know none of the math or the wire format.

Easy choice rejected: exposing a raw write(line) to the Lustre update loop. That leaks the handshake into the UI and makes every caller re-implement reliability. The narrow interface is what lets the Job FSM stay readable.

COMPOSITION

Small total transforms that recombine

The pipeline is a chain of total functions over values: parse → register → fit → apply → emit. Transform2D is a monoid-ish value you can compose (compose(bed_to_jig, jig_to_board)) and invert. Dry-run and real are the same generator with one field flipped (mode: DryRun | Drill, driving hover +0.2 vs plunge −2.5) — not two code paths.

// generation is one function, two configs — composition, not duplication.
// Gleam has no keyword args: the mode rides on the GcodeConfig record.
gcode_program.build(board, alignment, GcodeConfig(..cfg, mode: DryRun))  // spindle off, hover +0.2
gcode_program.build(board, alignment, GcodeConfig(..cfg, mode: Drill))   // M3 S255, plunge -2.5
STATE

One reference, everything else a value-across-time

Per P1, the board, the captured correspondences, the fitted transform, and the emitted program are all immutable values held in the Lustre model (the browser-side Elm-style state). The question "what did the operator capture, in what order?" is answerable because PendingAlignment.captured is an append-only list of values. The only managed reference is PrinterConnection, whose identity (the physical Web Serial port + Marlin's live head position) genuinely persists while its value changes.

Config resolves once: baud, bed size, tool diameters, feeds/depths — read at session start into an immutable config value (persisted in the browser's localStorage), never re-read mid-run.

INVARIANTS

The data model encodes the domain knowledge

This is the richest lens (see §00). The generative decision: holes live only in board space, and machine coordinates are a derived view through transform2d.apply. There is no field to desynchronize. The second: "not enough points yet" is a different Gleam type (PendingAlignment) than "a solved alignment" (Alignment), so no function that needs a transform can be handed an unsolved one. These are enforced by sum types: the legal graph is the set of case clauses, and the Aligned → Drilling shortcut literally does not type-check into existence. The Job FSM lifts these into legal sequencing — drilling has no representation before alignment.

ROBUSTNESS

Failures are loud, local, early — and physical

  • Parse edge: a malformed .drl or absolute-page-coordinate export (the X135/Y−149 trap) is rejected at board_model.parse with a diagnostic, not discovered as an out-of-bounds move mid-drill.
  • Residual gate: a bad fit (wrong points clicked, slipped board) shows a large residual and the drill button is unreachable. The misalignment is caught before the bit touches copper.
  • Explicit fault state: a disconnect mid-stream transitions PrinterConnection to Faulted, halts the stream, and surfaces in the UI — an explicit, recoverable state (recover via Reconnect), not a silent stall. There is no process to crash and no supervisor: the fault is a value in the state machine.
  • The energize snap is designed out (see invariants), the most physical failure of all.

Trade-off stated: we deliberately under-optimize persistence robustness — a crash loses the session (ephemeral, per non-goals). Acceptable because a session is minutes long and re-uploading is cheap; re-capturing 3 fiducials is the only redo cost.

03Progressive breakdown · BoardModel

The parsing edge

Interface: board_model.parse(Inputs) -> Result(BoardModel, ParseError). Everything downstream consumes a BoardModel and never touches a file.

// pure value — board coordinates only, never machine
pub type BoardModel {
  BoardModel(holes: List(Hole), outline: Outline,
             fiducials: List(Fiducial), tools: ToolTable, bbox: Bbox)
}

pub type Hole     { Hole(x: Float, y: Float, tool: ToolId) }       // board space
pub type Fiducial { Fiducial(x: Float, y: Float, kind: FiducialKind) } // Cross | Hole
type ToolTable = Dict(ToolId, Float)                          // tool_id -> diameter_mm
  • Depth: three different file formats (Excellon, RS-274X Gerber, edge SVG) collapse into one uniform value.
  • Robustness: the absolute-page-coordinate export is detected here (bbox far from origin) and rejected with guidance, not streamed.
  • Selectable points = fiducials ++ holes; the viz makes them clickable to start a Correspondence.

04Progressive breakdown · Alignment

The fitted transform

Interface (the smart constructor):

alignment.fit(List(Correspondence))
  -> Result(Alignment(transform: Transform2D, residuals: Residuals(rms, max)),
            FitError)
// FitError = TooFew     < 3 → caller still holds a PendingAlignment
//         | Degenerate  collinear / coincident points

transform2d.apply(t, #(bx, by)) -> #(mx, my)   // board → machine
transform2d.compose(a, b) / .invert(t)         // values compose
Why full affine, not G92: least-squares over 3–4 points yields translation + rotation + mirror + skew in one matrix. It absorbs the back-side X-mirror (no separate flag), a board not square to the bed (rotation), and the fiducial offset (translation) — the three things we juggled by hand. Residuals are the honesty check: they tell you the fit is trustworthy before you drill.

Composition note: a stable bed-to-jig transform (if a fixed fiducial jig is ever added) composes with the per-board jig-to-board transform — the type already supports it, so the ephemeral-vs-calibration choice can be revisited without reshaping the math.

05Progressive breakdown · PrinterConnection

The one identity

The serial control state machine owning the Web Serial port (navigator.serial). It is the system's only mutable reference and its deepest component, living in src/blau_drill/control/: a pure transition core (printer.gleam: command(state, Command) / feed(state, line) → Step(state, writes, events)) wrapped by a thin effectful shell (controller.gleam) over a Backend seam — transport.web_serial() for the real port, transport.simulator() for hardware-free dev. It is not a process and there is no supervisor: state is a plain value. Operators drive it with the command verbs:

Where                  // query live position (M114); valid in Idle/Jogging
Energize / Release     // M17 / M18 — the only entry to / exit from Jogging
Jog(axis, mm) / MoveTo(x, y)   // relative / absolute motion — only in Jogging
Stream(lines)         // stream a program; ok-handshake inside
Halt / Reconnect       // M112-class abort → Faulted; recover Faulted → Idle
stateDiagram-v2 [*] --> Disconnected Disconnected --> Idle: connect (Web Serial) Idle --> Jogging: energize (M17) Jogging --> Idle: release (M18) Jogging --> Streaming: begin program Idle --> Streaming: begin program Streaming --> Idle: program done Streaming --> Faulted: serial loss / halt Faulted --> Idle: reconnect note right of Idle: no jog command matches here\n(designs out the 1–2 mm snap)
  • Depth: the entire reliability protocol (line numbers, checksums, resend) is invisible to callers.
  • Robustness: Faulted is reachable from any active state and is loud; Reconnect recovers cleanly to Idle — an explicit transition, not a process restart.

06Progressive breakdown · GcodeProgram

Native generation

Interface: gcode_program.build(board, alignment, cfg) -> GcodeProgram(lines, mode, bbox_machine, tool_order). Requires an Alignment by type — there is no way to generate from unaligned holes.

// Per hole: transform, then emit. No pcb2gcode, no Python.
let #(mx, my) = transform2d.apply(alignment.transform, #(hole.x, hole.y))
// Tool-grouped, M3 S255 (laser-PWM spindle), dwell, plunge per mode.
DryRun -> spindle off, Z +0.2   // registration check
Drill  -> M3 S255,   Z -2.5   // real, into sacrificial layer

This folds postprocess_drill.py's three jobs (spindle fix, dry-run variant, fiducial offset) into typed parameters — the mode field of the GcodeConfig record. The Marlin spindle quirk (S must be on the M3 line) is a property of this module, encoded once.

07Progressive breakdown · Job

The state machine that makes the path legal

stateDiagram-v2 [*] --> Parsed: upload accepted Parsed --> Registering: pick board points Registering --> Registering: capture correspondence Registering --> Aligned: fit ok & residuals ≤ tol Registering --> AlignmentRejected: residuals > tol AlignmentRejected --> Registering: recapture Aligned --> DryRun: run dry-run DryRun --> Aligned: residuals look wrong, redo DryRun --> Drilling: confirm registration Drilling --> Done: all holes Drilling --> Faulted: serial loss Faulted --> Aligned: reconnect & resume

The FSM is the encapsulation the brief asked for. It is a Gleam sum type with a pure transition(job, event) -> Result(Job, _): each event exists only in the states where it's legal — no "drill" in Parsed, no Aligned → Drilling case clause (must pass DryRun). The Lustre view renders the current state and overlays the live head position on the board canvas (the Lustre SVG view).

08End-to-end usage walkthrough

One board, start to finish

  1. Open & connect. The operator opens the app URL in Chromium (no install) and clicks "Connect" — a user gesture that opens the Web Serial port chooser. Drop the KiCad .drl + Edge.Cuts + back-copper exports. board_model.parse runs; the Lustre SVG canvas draws the outline, holes (colored by tool), and fiducial crosses. Parsed
  2. Pick registration points. Click 3–4 fiducials (or distinctive holes) on the canvas. Each becomes a pending Correspondence awaiting its machine coordinate. Registering
  3. Energize & jog. Hit "Enable steppers" (Energize — the app refuses to Jog from Idle, designs out the snap). Use on-screen jog buttons to drive the bit onto the first physical fiducial.
  4. Capture. Click "Capture" → app issues Where (reads M114), pairs it with the selected board point. Repeat for each point. The canvas shows captured-vs-pending.
  5. Fit. On the 3rd non-collinear capture, alignment.fit solves the affine and shows residuals. Green if under tolerance → Aligned; red → AlignmentRejected, recapture.
  6. Dry-run. Stream gcode_program.build(.., GcodeConfig(.., mode: DryRun)) — spindle off, bit hovers 0.2 mm over every hole. Watch the head trace the pattern on the real board and on the Lustre SVG canvas in real time. DryRun
  7. Drill. Confirm registration looked right → stream with mode: Drill. M3 S255, plunge −2.5, per-tool pauses for bit changes. Drilling → Done

Everything we did by hand this session — re-export with the right origin, no-mirror, fiducial G92, post-process for M3 S255, dry-run first — becomes a single guided flow where the wrong order is unrepresentable.

09Glossary

Domain vocabulary

BoardModel
Immutable parse of the KiCad outputs: holes, outline, fiducials, tools — all in board coordinates.
Correspondence
A captured pair {board_point, machine_point} — a board feature and where the printer actually is when the operator locates it.
PendingAlignment
The in-progress type holding <3 correspondences. Has no transform field — cannot be used to drill.
Alignment
A solved affine Transform2D plus its residuals. Constructible only via alignment.fit from ≥3 non-collinear points.
Transform2D
A 2×3 affine matrix mapping board → machine. A composable, invertible value.
Residuals
Per-point fit error (rms + max). The trust signal; gates the real run.
PrinterConnection
The single stateful identity: the serial control state machine (a pure printer.gleam core + an effectful controller.gleam shell over a Backend seam) owning the Web Serial port, the Marlin handshake, live position, and mode (Disconnected | Idle | Jogging | Streaming | Faulted).
GcodeProgram
Generated Marlin G-code for one mode (DryRun | Drill), built from an Alignment.
Job
The session state machine enforcing the legal order: parse → register → align → dry-run → drill.
