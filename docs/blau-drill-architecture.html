00Goals, principles & invariants

The foundation

Goals

  • Take a KiCad board through one drilling session, end to end, in one app: load → register → align → dry-run → drill.
  • Replace the manual chain (re-export, mirror flag, fiducial G92, pcb2gcode, postprocess_drill.py) with a single fitted affine transform that absorbs offset, rotation, mirror, and skew.
  • Give a human a visual, jog-driven alignment loop with live position feedback and trustworthy residuals.
  • Generate Marlin-correct G-code natively (spindle on shared laser PWM as M3 S255).

Non-goals load-bearing

  • Not a multi-user / multi-machine service. One bench, one printer, one operator.
  • Not a persistent job database — each session is a fresh upload (ephemeral state).
  • Not a CAM suite. No trace isolation, no laser-ablation pass — drilling only (for now).
  • Not a replacement for OctoPrint as a general print host — it owns the serial port only for the duration of a drilling session.

Core principles

P1
The board is a value; the printer is the only identity. Parsed boards, transforms, and G-code are immutable values that flow through pure functions. Exactly one stateful reference exists: the serial PrinterConnection.
P2
Alignment is a solved transform, not a procedure. Registration captures correspondences; the transform is fitted and carries its own residuals. No G92 ritual in the domain — that's an implementation detail of streaming.
P3
Every boundary parses into a precise type. The .drl, Gerber, and each Marlin serial line are parsed at the edge into domain values; malformed input fails loudly there, never deep inside.
P4
Illegal sequencing is unrepresentable. You cannot stream a drill without an Alignment; you cannot fit an Alignment without ≥3 non-collinear points. The Job state machine makes the only legal path the only typed path.
P5
The dangerous defaults are enforced, not documented. Energize-before-jog, residual gate before drilling, dry-run before real — each is a state the machine must pass through, not a README sentence.

Invariants — each paired with the representation that enforces it

Invariant (must always hold)Made unrepresentable by…
An Alignment exists only when solvable. No public constructor. Only Alignment.fit(corrs) returns {:ok, %Alignment{}}, and only for ≥3 non-collinear points; otherwise {:error, :degenerate}. Fewer points are a PendingAlignment — a different struct.
Drilling cannot start without a valid alignment. GcodeProgram.build/2 takes an %Alignment{} as a required argument — there is no arity that accepts raw holes. The "Drill" event only exists in Job state :aligned.
Residuals over tolerance block the real run. Job transition :aligned → :drilling is guarded by residuals.max ≤ tol; a failing fit lands in :alignment_rejected, which has no drill event.
You never jog a de-energized axis (the 1–2 mm snap). PrinterConnection mode :idle exposes no jog command. The only path to :jogging runs an energize+settle step first.
Real drilling is never the first thing that runs. Job has no edge from :aligned straight to :drilling — it routes through :dry_run, whose completion is the precondition for the real-run event.
A hole's machine coordinate is always the transform of its board coordinate. Holes are stored only in board space. Machine coordinates are computed on demand by Transform2D.apply/2 — there is no settable machine-coordinate field to drift out of sync.

The central type lattice

The registration → alignment progression is the spine. Illegal states (a "drilling job with no transform", a "transform from 2 points") have no node — they cannot be constructed.

02Lens-driven design rationale

Why it's shaped this way

DEPTH

Two deep modules carry the hard parts

PrinterConnection hides the entire Marlin serial protocol — port lifecycle, line numbering, checksums, the ok/resend handshake, M114 polling, flow control — behind four verbs: jog/2, where/1, stream/2, halt/1. Alignment hides a least-squares affine solve behind one function: fit(correspondences). A caller needs to know none of the math or the wire format.

Easy choice rejected: exposing raw send_gcode(line) to LiveView. That leaks the handshake into the UI and makes every caller re-implement reliability. The narrow interface is what lets the Job FSM stay readable.

COMPOSITION

Small total transforms that recombine

The pipeline is a chain of total functions over values: parse → register → fit → apply → emit. Transform2D is a monoid-ish value you can compose (compose(bed_to_jig, jig_to_board)) and invert. Dry-run and real are the same generator with one parameter flipped (plunge: {:hover, 0.2} | {:drill, -2.5}) — not two code paths.

# generation is one function, two configs — composition, not duplication
GcodeProgram.build(alignment, mode: :dry_run)   # spindle off, hover +0.2
GcodeProgram.build(alignment, mode: :drill)     # M3 S255, plunge -2.5
STATE

One reference, everything else a value-across-time

Per P1, the board, the captured correspondences, the fitted transform, and the emitted program are all immutable values held in LiveView assigns. The question "what did the operator capture, in what order?" is answerable because PendingAlignment.captured is an append-only list of values. The only managed reference is PrinterConnection, whose identity (the physical port + Marlin's live head position) genuinely persists while its value changes.

Config resolves once: serial port, baud, bed size, tool diameters, feeds/depths — read at session start into an immutable Session.config, never re-read mid-run.

INVARIANTS

The data model encodes the domain knowledge

This is the richest lens (see §00). The generative decision: holes live only in board space, and machine coordinates are a derived view through Transform2D.apply. There is no field to desynchronize. The second: "not enough points yet" is a different type (PendingAlignment) than "a solved alignment" (Alignment), so no function that needs a transform can be handed an unsolved one. The Job FSM lifts these into legal sequencing — drilling literally has no representation before alignment.

ROBUSTNESS

Failures are loud, local, early — and physical

  • Parse edge: a malformed .drl or absolute-page-coordinate export (the X135/Y−149 trap) is rejected at BoardModel.parse with a diagnostic, not discovered as an out-of-bounds move mid-drill.
  • Residual gate: a bad fit (wrong points clicked, slipped board) shows a large residual and the drill button is unreachable. The misalignment is caught before the bit touches copper.
  • Serial supervision: a disconnect mid-stream transitions PrinterConnection to :faulted, halts the stream, and surfaces in the UI — the GenServer crash is local and supervised, not a silent stall.
  • The energize snap is designed out (see invariants), the most physical failure of all.

Trade-off stated: we deliberately under-optimize persistence robustness — a crash loses the session (ephemeral, per non-goals). Acceptable because a session is minutes long and re-uploading is cheap; re-capturing 3 fiducials is the only redo cost.

03Progressive breakdown · BoardModel

The parsing edge

Interface: BoardModel.parse(%{drl:, edge_cuts:, copper:}) :: {:ok, %BoardModel{}} | {:error, reason}. Everything downstream consumes a BoardModel and never touches a file.

defmodule BoardModel do
  # pure value — board coordinates only, never machine
  defstruct [:holes, :outline, :fiducials, :tools, :bbox]
end

Hole      :: %{x: float, y: float, tool: tool_id}        # board space
Fiducial  :: %{x: float, y: float, kind: :cross | :hole}  # registration candidate
ToolTable :: %{tool_id => diameter_mm}
  • Depth: three different file formats (Excellon, RS-274X Gerber, edge SVG) collapse into one uniform value.
  • Robustness: the absolute-page-coordinate export is detected here (bbox far from origin) and rejected with guidance, not streamed.
  • Selectable points = fiducials ++ holes; the viz makes them clickable to start a Correspondence.

04Progressive breakdown · Alignment

The fitted transform

Interface (the smart constructor):

Alignment.fit([Correspondence]) ::
    {:ok, %Alignment{transform: %Transform2D{}, residuals: %{rms, max}}}
  | {:error, :too_few}      # < 3 → caller still holds a PendingAlignment
  | {:error, :degenerate}   # collinear / coincident points

Transform2D.apply(t, {bx, by}) :: {mx, my}   # board → machine
Transform2D.compose(a, b) / .invert(t)        # values compose
Why full affine, not G92: least-squares over 3–4 points yields translation + rotation + mirror + skew in one matrix. It absorbs the back-side X-mirror (no separate flag), a board not square to the bed (rotation), and the fiducial offset (translation) — the three things we juggled by hand. Residuals are the honesty check: they tell you the fit is trustworthy before you drill.

Composition note: a stable bed-to-jig transform (if a fixed fiducial jig is ever added) composes with the per-board jig-to-board transform — the type already supports it, so the ephemeral-vs-calibration choice can be revisited without reshaping the math.

05Progressive breakdown · PrinterConnection

The one identity

A supervised GenServer owning the serial port (via circuits_uart). It is the system's only mutable reference and its deepest module.

PrinterConnection.where(conn)            :: {:ok, {x, y, z}}        # M114
PrinterConnection.jog(conn, axis, mm)     :: :ok | {:error, :idle}  # only in :jogging
PrinterConnection.stream(conn, program)   :: :ok                     # ok-handshake inside
PrinterConnection.halt(conn)               :: :ok                     # M112-class abort
stateDiagram-v2 [*] --> idle idle --> jogging: energize + settle jogging --> idle: release jogging --> streaming: begin program idle --> streaming: begin program streaming --> idle: program done streaming --> faulted: serial loss / halt faulted --> idle: reconnect note right of idle: no jog command exposed here\n(designs out the 1–2 mm snap)
  • Depth: the entire reliability protocol (line numbers, checksums, resend) is invisible to callers.
  • Robustness: :faulted is reachable from any active state and is loud; the supervisor restarts cleanly to :idle.

06Progressive breakdown · GcodeProgram

Native generation

Interface: GcodeProgram.build(%Alignment{}, opts) :: %GcodeProgram{lines, mode, bbox_machine}. Requires an Alignment by type — there is no way to generate from unaligned holes.

# Per hole: transform, then emit. No pcb2gcode, no Python.
machine_xy = Transform2D.apply(alignment.transform, hole)
# Tool-grouped, M3 S255 (laser-PWM spindle), dwell, plunge per mode.
mode: :dry_run -> spindle off, Z +0.2   # registration check
mode: :drill   -> M3 S255,   Z -2.5   # real, into sacrificial layer

This folds postprocess_drill.py's three jobs (spindle fix, dry-run variant, fiducial offset) into typed parameters. The Marlin spindle quirk (S must be on the M3 line) is a property of this module, encoded once.

07Progressive breakdown · Job

The state machine that makes the path legal

stateDiagram-v2 [*] --> parsed: upload accepted parsed --> registering: pick board points registering --> registering: capture correspondence registering --> aligned: fit ok & residuals ≤ tol registering --> alignment_rejected: residuals > tol alignment_rejected --> registering: recapture aligned --> dry_run: run dry-run dry_run --> aligned: residuals look wrong, redo dry_run --> drilling: confirm registration drilling --> done: all holes drilling --> faulted: serial loss faulted --> aligned: reconnect & resume

The FSM is the encapsulation the brief asked for. Each event exists only in the states where it's legal: no "drill" in :parsed, no straight edge from :aligned to :drilling (must pass :dry_run). LiveView renders the current state; live_svelte overlays live head position on the board canvas.

08End-to-end usage walkthrough

One board, start to finish

  1. Upload. Drop the KiCad .drl + Edge.Cuts + back-copper exports. BoardModel.parse runs; the svelte canvas draws the outline, holes (colored by tool), and fiducial crosses. parsed
  2. Pick registration points. Click 3–4 fiducials (or distinctive holes) on the canvas. Each becomes a pending Correspondence awaiting its machine coordinate. registering
  3. Energize & jog. Hit "Enable steppers" (the app refuses to jog from idle — designs out the snap). Use on-screen jog buttons to drive the bit onto the first physical fiducial.
  4. Capture. Click "Capture" → app reads M114, pairs it with the selected board point. Repeat for each point. The canvas shows captured-vs-pending.
  5. Fit. On the 3rd non-collinear capture, Alignment.fit solves the affine and shows residuals. Green if under tolerance → aligned; red → alignment_rejected, recapture.
  6. Dry-run. Stream GcodeProgram.build(mode: :dry_run) — spindle off, bit hovers 0.2 mm over every hole. Watch the head trace the pattern on the real board and on the canvas in real time. dry_run
  7. Drill. Confirm registration looked right → stream mode: :drill. M3 S255, plunge −2.5, per-tool pauses for bit changes. drilling → done

Everything we did by hand this session — re-export with the right origin, no-mirror, fiducial G92, post-process for M3 S255, dry-run first — becomes a single guided flow where the wrong order is unrepresentable.

09Glossary

Domain vocabulary

BoardModel
Immutable parse of the KiCad outputs: holes, outline, fiducials, tools — all in board coordinates.
Correspondence
A captured pair {board_point, machine_point} — a board feature and where the printer actually is when the operator locates it.
PendingAlignment
The in-progress type holding <3 correspondences. Has no transform field — cannot be used to drill.
Alignment
A solved affine Transform2D plus its residuals. Constructible only via fit/1 from ≥3 non-collinear points.
Transform2D
A 2×3 affine matrix mapping board → machine. A composable, invertible value.
Residuals
Per-point fit error (rms + max). The trust signal; gates the real run.
PrinterConnection
The single stateful GenServer: serial port, Marlin handshake, live position, mode (:idle/:jogging/:streaming/:faulted).
GcodeProgram
Generated Marlin G-code for one mode (:dry_run | :drill), built from an Alignment.
Job
The session state machine enforcing the legal order: parse → register → align → dry-run → drill.
