blau-drill

A single-operator, pure-browser instrument (Gleam → JavaScript via Lustre, no backend, no install) that loads a KiCad board, drives the modified Two Trees Bluer over the Web Serial API, fits an affine board → machine transform from human-located fiducials, and streams the drilling — dry-run first. One Session coordinates the stage, the wire, and the screen.

00Goals, principles & invariants

The foundation

Goals

  • Take a KiCad board through one drilling session, end to end, in one app: load → register → align → dry-run → drill.
  • Replace the manual chain (re-export, mirror flag, fiducial G92, pcb2gcode, postprocess_drill.py) with a single fitted affine transform that absorbs offset, rotation, mirror, and skew.
  • Give a human a visual, jog-driven alignment loop with live position feedback and trustworthy residuals.
  • Generate Marlin-correct G-code natively (spindle on shared laser PWM as M3 S255).
  • Coordinate the stage, the wire, and the screen as one value — so the UI cannot show a screen the machines contradict and a cross-machine action is atomic. coordination

Non-goals load-bearing

  • Not a multi-user / multi-machine service. One bench, one printer, one operator.
  • Not a persistent job database — each session is a fresh upload (ephemeral state, ADR-0004).
  • Not a CAM suite. No trace isolation, no laser-ablation pass — drilling only (for now).
  • Not a replacement for OctoPrint as a general print host — it owns the serial port only for the duration of a drilling session.
  • Not a state-machine library. Every machine — including the new Session — is a plain Gleam sum type with a pure transition function (ADR-0007).

Runtime constraints come with the architecture

  • No backend, no install — the win: the whole app is Gleam compiled to JavaScript (Lustre), served as static files. The operator just opens a URL; there is no server to run, no server-side rendering, no database. All state lives in the Lustre model in the browser.
  • Chromium-only: it talks to the printer directly over the Web Serial API (navigator.serial), available in Chrome / Edge / Opera 89+. Firefox and Safari are not supported.
  • Secure context required: Web Serial only works over HTTPS, localhost, or file://.
  • User-gesture connect: the port chooser can only open from an explicit operator click — the app can never silently grab the port.

Core principles

P1
The board is a value; the printer is the only identity. Parsed boards, transforms, and G-code are immutable values that flow through pure functions. Exactly one stateful reference exists: the serial PrinterConnection. State
P2
Alignment is a solved transform, not a procedure. Registration captures correspondences; the transform is fitted and carries its own residuals. No G92 ritual in the domain — that's an implementation detail of streaming. Invariants
P3
Every boundary parses into a precise type. The .drl, Gerber, and each Marlin serial line are parsed at the edge into domain values; malformed input fails loudly there, never deep inside. Robustness
P4
Illegal sequencing is unrepresentable. You cannot stream a drill without an Alignment; you cannot fit one without ≥3 non-collinear points; and you cannot start a drill while a stream is in flight — the Job and Session machines make the only legal path the only typed path. Invariants
P5
The dangerous defaults are enforced, not documented. Energize-before-jog, residual gate before drilling, dry-run before real, and abort stops motion — each is a state the machine must pass through, not a README sentence. Robustness
P6
One coordinator owns cross-machine moves; the UI is projected. coordination The Session nests the real job and printer values (never a copy), the screen is a pure projection of it, and every flow action is one transition returning an ordered Plan. There is nothing to keep in sync. StateDepth

Invariants — each paired with the representation that enforces it

Invariant (must always hold)Made unrepresentable by…
An Alignment exists only when solvable. No public constructor. Only alignment.fit(corrs) returns Ok(Alignment(..)), and only for ≥3 non-collinear points; otherwise Error(Degenerate). Fewer points are a PendingAlignment — a different Gleam type.
Drilling cannot start without a valid alignment. gcode_program.build takes an Alignment as a required argument — there is no overload that accepts raw holes. The "Drill" event only exists in Job state Aligned.
Residuals over tolerance block the real run. Job transition Aligned → Drilling is guarded by residuals.max ≤ tol; a failing fit lands in AlignmentRejected, which has no drill event.
You never jog a de-energized axis (the 1–2 mm snap). PrinterConnection state Idle matches no Jog command — the pure core returns a NotEnergized refusal and writes nothing. The only path to Jogging runs an energize step (M17) first.
Real drilling is never the first thing that runs. Job has no Aligned → Drilling case clause — it routes through DryRun, whose completion is the precondition for the real-run event.
A hole's machine coordinate is always the transform of its board coordinate. Holes are stored only in board space. Machine coordinates are computed on demand by transform2d.apply — there is no settable machine-coordinate field to drift out of sync.
There is exactly one stage value and one wire value in the app. new The Session variants contain job.Job / printer.PrinterState. model.screen, model.PrinterState, and the bridge.printer_state mirror are deleted.
The UI screen always matches the FSMs. new screen is computed by session.screen/2; there is no settable screen field to drift. (ADR-0012)
A drill stream cannot start while another stream is in flight. new The Drilling session variant is constructed only by the transition that also emits the cancel-then-stream Plan.
Abort stops motion, not just the controller. new Abort is one Session transition whose Plan is Halt + queue-clear; the emulator's motion queue lets a test assert the head stops. (ADR-0013)

01System at a glance

One coordinator over five pure modules and one identity

Files enter on the left as values; the only thing with identity is the serial link to Marlin. Above the two machines sits the Session — a value derived on demand from the model's one Job (stage) and one PrinterConnection (wire) via session.of(job, board, wire). It nests those real values (never copies them), projects the screen the views render, and on a flow action returns an ordered Plan the app executes through the controller. Recomputing it each frame (rather than storing it) means it can never drift from the machines it is derived from.

02Lens-driven design rationale

Why it's shaped this way

DEPTH

Deep modules carry the hard parts behind small interfaces

PrinterConnection hides the entire Marlin serial protocol — port lifecycle, line numbering, checksums, the ok/resend handshake, M114 polling, flow control — behind a small set of verbs. Alignment hides a least-squares affine + Z-plane solve behind one function, alignment.fit. And the new Session hides all cross-machine choreography — cancel-then-drill, reconcile-on-abort, route-a-serial-loss-into-both-machines — behind just transition/2 and screen/2.

Easy choice rejected: letting each app.update handler poke the job, the printer, and the screen independently. That is precisely the three-authority glue that produced the dry-run→drill bug. The narrow coordinator interface is what makes the cross-machine ordering legal by construction.

COMPOSITION

Small total transforms that recombine

The pipeline is a chain of total functions over values: parse → register → fit → apply → emit. Dry-run and real are the same generator with one field flipped. The Plan the coordinator returns is just List(printer.Command) — it composes with the controller the app already has, so the coordinator reuses every wire transition rather than duplicating it. And the faithful emulator is one pure core with two drivers: a hand-pumped tick for deterministic tests and an auto-pump FFI shim for a live in-app virtual machine.

// generation is one function, two configs — composition, not duplication.
gcode_program.build(board, alignment, GcodeConfig(..cfg, mode: DryRun))  // spindle off, hover +0.2
gcode_program.build(board, alignment, GcodeConfig(..cfg, mode: Drill))   // M3 S255, plunge -2.5
STATE

One reference, one session value, everything else a value-across-time

Per P1, the board, the captured correspondences, the fitted transform, and the emitted program are immutable values in the Lustre model. The only managed reference is PrinterConnection. The drift we removed (P6) had two literal roots: model.PrinterState (model.gleam:117) was a lossy mirror of printer.PrinterState bridged by hand, and screen was a stored copy of f(job, printer). The cure is structural: one Session value is the spine; it contains the single job and printer; the screen and the mirror are deleted.

Config resolves once: baud, bed size, tool diameters, feeds/depths — read at session start into an immutable config value (persisted in localStorage), never re-read mid-run.

INVARIANTS

The data model encodes the domain knowledge

This is the richest lens (see §00). The generative decisions: holes live only in board space (machine coords are a derived view); "not enough points yet" is a different Gleam type than "a solved alignment"; and now "drilling" is a Session variant constructed only with its own in-flight stream, so "Drilling but nothing streaming" — the exact bug — has no representation. Each is the same move: make the illegal combination fail to type-check, the way PendingAlignment vs Alignment already does.

ROBUSTNESS

Failures are loud, local, early — and physical

  • Parse edge: a malformed .drl or absolute-page-coordinate export is rejected at board_model.parse with a diagnostic, not discovered as an out-of-bounds move mid-drill.
  • Residual gate: a bad fit shows a large residual and the drill button is unreachable — the misalignment is caught before the bit touches copper.
  • Explicit fault state: a disconnect mid-stream transitions to Faulted, halts the stream, and surfaces in the UI — an explicit, recoverable value, not a silent stall.
  • Rejected flow actions: an illegal Session action returns a typed Rejected(reason) and writes nothing — never a half-applied cross-machine transition. new
  • Abort stops motion: the emulator's motion queue makes "still moving after abort" a concrete, assertable state, and Halt clears it — turning a real-hardware safety hazard into a regression test. new
  • Envelope limits: an out-of-bounds / past-min move is a Marlin-style error reply with the head unmoved. new

Trade-off stated: we deliberately under-optimize persistence robustness — a crash loses the session (ephemeral, per non-goals). Acceptable because a session is minutes long and re-capturing 3 fiducials is the only redo cost. Per ADR-0011, that loss is also a safety feature: no trusted position survives a motors-off gap.

03Progressive breakdown · Session new

The coordinator that owns stage + wire + screen

Interface: session.transition(Session, Action) -> Result(#(Session, Plan), Rejected) and session.screen(Session, Overlay) -> Screen. Pure, in ui/session.gleam — the coordination tier above both control and the ui model (it nests printer.PrinterState from control and projects to model.Screen, so it cannot live in domain/, which imports neither). It mirrors the printer core's Step(state, writes, events) shape one level up. Each variant carries the actual job.Job / printer.PrinterState, never a copy.

// ui/session.gleam — the outer coordinator value (pure, no IO)
pub type Session {
  Loading(board: BoardOpt, printer: printer.PrinterState)   // pre-parse / pre-connect
  Aligning(job: job.Job, printer: printer.PrinterState)     // Registering|Aligned|Rejected
  Rehearsing(job: job.Job, printer: printer.PrinterState)   // DryRun — stream REQUIRED
  Drilling(job: job.Job, printer: printer.PrinterState)     // constructed ONLY with a drill plan
  Completed(job: job.Job, printer: printer.PrinterState)    // Done
  Faulted(job: JobOpt, printer: printer.PrinterState)        // off-ramp from any active state
}
pub type Overlay { NoOverlay  SettingsOpen  LogOpen }  // side routes — orthogonal to lifecycle
pub type Plan = List(printer.Command)             // ordered; app runs it in ONE effect
pub type Rejected { IllegalHere(Action)  JobRefused(job.TransitionError)  WireBusy }
stateDiagram-v2 [*] --> Loading Loading --> Aligning: StartRegistering (energized) Aligning --> Aligning: Capture / Fit / RestartAlignment Aligning --> Rehearsing: RunDryRun ⟹ Plan[Stream dry-run] Rehearsing --> Aligning: RedoAlignment ⟹ Plan[CancelStream] Rehearsing --> Drilling: ConfirmRegistration ⟹ Plan[CancelStream, Stream drill] Drilling --> Completed: MarkComplete (stream done) Aligning --> Loading: Deenergize (ADR-0011 reset) Rehearsing --> Loading: Deenergize Drilling --> Faulted: Abort ⟹ Plan[Halt + queue-clear] Rehearsing --> Faulted: Abort / SerialLost Aligning --> Faulted: SerialLost Faulted --> Loading: Reconnect (re-register) Completed --> [*]
  • Nest, don't copy: exactly one job and one printer exist; model.screen / model.PrinterState / bridge.printer_state are deleted. Nothing to sync, nothing to drift.
  • Project, don't store: screen(session, overlay) is total; the side routes override it without becoming lifecycle states that clobber the job.
  • Atomic cross-machine moves: Rehearsing → Drilling is one transition returning Plan[CancelStream, Stream(drill)] — the drill cannot be refused Busy because the cancel precedes it in the same effect.
The fix, on one edge. The old three-write confirm_registration handler (advance job + set screen + stream, un-atomic) collapses into a single legal transition. See ADR-0012.

04Progressive breakdown · BoardModel

The parsing edge

Interface: board_model.parse(Inputs) -> Result(BoardModel, ParseError). Everything downstream consumes a BoardModel and never touches a file.

// pure value — board coordinates only, never machine
pub type BoardModel {
  BoardModel(holes: List(Hole), outline: Outline,
             fiducials: List(Fiducial), tools: ToolTable, bbox: Bbox)
}
pub type Hole     { Hole(x: Float, y: Float, tool: ToolId) }       // board space
pub type Fiducial { Fiducial(x: Float, y: Float, kind: FiducialKind) } // Cross | Hole
type ToolTable = Dict(ToolId, Float)                          // tool_id -> diameter_mm
  • Depth: three different file formats (Excellon, RS-274X Gerber, edge SVG) collapse into one uniform value.
  • Robustness: the absolute-page-coordinate export is detected here (bbox far from origin) and rejected with guidance, not streamed.
  • Selectable points = fiducials ++ holes; the viz makes them clickable to start a Correspondence.

05Progressive breakdown · Alignment

The fitted transform (2.5D)

Interface (the smart constructor):

alignment.fit(List(Correspondence))
  -> Result(Alignment(transform: Transform2D, plane: ZPlane, residuals: Residuals(rms, max)),
            FitError)
// FitError = TooFew     < 3 → caller still holds a PendingAlignment
//         | Degenerate  collinear / coincident points
transform2d.apply(t, #(bx, by)) -> #(mx, my)   // board → machine (XY)
Why full affine + a Z plane, not G92: least-squares over 3–4 points yields translation + rotation + mirror + skew in one matrix and a tilted surface plane z = a·bx + b·by + c (ADR-0010), so drill depth references the real per-hole surface. It absorbs the back-side X-mirror (no separate flag), a board not square to the bed, and the fiducial offset. Residuals are the honesty check — they tell you the fit is trustworthy before you drill.

06Progressive breakdown · PrinterConnection

The one identity

The serial control state machine owning the Web Serial port. It is the system's only mutable reference, living in src/blau_drill/control/: a pure transition core (printer.gleam: command(state, Command) / feed(state, line) → Step(state, writes, events)) wrapped by a thin effectful shell (controller.gleam) over a Backend seam — transport.web_serial() for the real port, transport.simulator() or the faithful emulator for hardware-free dev. State is a plain value; there is no process. The Session holds this value directly.

Jog(axis, mm) / MoveTo(x, y, z_lift)   // relative / safe-jump motion — only in Jogging

  • Robustness: Faulted is reachable from any active state and is loud; Reconnect recovers cleanly — an explicit transition, not a restart. Halt / CancelStream stay reachable from StreamPaused so abort is never gated behind a pause.

07Progressive breakdown · GcodeProgram

Native generation

Interface: gcode_program.build(board, alignment, cfg) -> GcodeProgram(lines, mode, bbox_machine, tool_order). Requires an Alignment by type — there is no way to generate from unaligned holes.

// Per hole: transform, then emit. No pcb2gcode, no Python.
let #(mx, my) = transform2d.apply(alignment.transform, #(hole.x, hole.y))
// Tool-grouped, M3 S255 (laser-PWM spindle), dwell, plunge per mode.
DryRun -> spindle off, Z +0.2   // registration check
Drill  -> M3 S255,   Z -2.5   // real, into sacrificial layer

The streamed form is sanitized (no blank/comment lines — Marlin does not ok a blank) and, with app_pause on, pauses in-app at each bit-change boundary rather than emitting M0 (ADR-0009). The Marlin spindle quirk (S must be on the M3 line) is encoded once here.

08Progressive breakdown · Job

The stage machine the Session embeds

stateDiagram-v2 [*] --> Parsed: upload accepted Parsed --> Registering: pick board points Registering --> Aligned: fit ok & residuals ≤ tol Registering --> AlignmentRejected: residuals > tol AlignmentRejected --> Registering: recapture Aligned --> DryRun: run dry-run DryRun --> Aligned: residuals look wrong, redo DryRun --> Drilling: confirm registration Drilling --> Done: all holes Drilling --> Faulted: serial loss Faulted --> Parsed: reconnect (re-register · ADR-0011)

The Job FSM is a Gleam sum type with a pure transition(job, event) -> Result(Job, _): each event exists only where it's legal — no "drill" in Parsed, no Aligned → Drilling case clause. Per ADR-0011, any de-energize (release, fault, serial loss, refresh) discards the alignment and routes back to a clean slate — no trusted position survives a motors-off gap. The Session embeds this value and lifts its events into the cross-machine plan.

09Progressive breakdown · Faithful emulator new

A motion queue, an envelope, and time

The Marlin emulator (control/marlin_emulator.gleam) is a pure protocol core: it validates line numbers + checksums, tracks motor state, blocks on M0, and answers M114. It is leveled up with physical time (a planner queue drained by a tick) and an envelope, so the safety-relevant states the thin simulator masks become observable.

pub type EmulatorState {
  EmulatorState(last_line: Int, motors_on: Bool, abs: Bool, x: Float, y: Float, z: Float, paused: Bool,
    queue: List(QueuedMove),   // NEW — admitted-but-not-yet-executed moves (the planner buffer)
    bounds: Bounds)             // NEW — injected XYZ envelope (test-supplied, never a product default)
}
pub fn feed(s, line) -> #(EmulatorState, List(String))  // admits a move + acks admission
pub fn tick(s, dt) -> EmulatorState      // drains the queue; SEPARATE from feed so a test can
                                  // feed + NOT tick → assert the head is still mid-move
pub fn halt(s) -> EmulatorState           // clears the queue → motion stops (assert queue == [])
pub fn force(fields…) -> EmulatorState     // test seam: drop into ANY condition directly

Envelope limits Robustness

A move whose target is outside bounds (or past a negative min) replies a Marlin-style error and is not admitted — the head does not advance. Bounds are injected at construction (tests pick the envelope), so we add no product default — motion limits are operator/hardware config (ADR-0004).

Abort-mid-move State

Feed a long move, do not tick: the queue is non-empty and the head sits at its start — the "still moving" state. Issue Halt: the queue empties. The e2e test asserts both, turning "physical motion continues after abort" into a regression test.

10End-to-end usage walkthrough

One board, start to finish

  1. Open & connect. Open the app URL in Chromium (no install) and click "Connect" — a user gesture that opens the Web Serial port chooser. Drop the KiCad .drl + Edge.Cuts + back-copper exports. board_model.parse runs; the canvas draws outline, holes, and fiducials. The Session is Loading; screen projects Load.
  2. Pick registration points. Click 3–4 fiducials (or distinctive holes). Each becomes a pending Correspondence. Session Aligning → Align.
  3. Energize & jog. Hit "Enable steppers" (Energize — the app refuses to Jog from Idle, designs out the snap). Jog the bit onto the first physical fiducial.
  4. Capture. "Capture" → app issues Where (M114), pairs it with the selected board point and its machine Z. Repeat for each point.
  5. Fit. On the 3rd non-collinear capture, alignment.fit solves the affine + Z plane and shows residuals. Green → Align (ready); red → recapture.
  6. Dry-run. "Run dry-run" → Action: RunDryRun → Session Rehearsing, Plan = [Stream(dry-run)]. Spindle off, hover +0.2 over every hole. DryRun
  7. Confirm → Drill. "Confirm" → Action: ConfirmRegistration → Session Drilling, Plan = [CancelStream, Stream(drill)] run in one ordered effect: the dry-run cancels, the drill streams cleanly. screen projects Drill only now — and it's true. M3 S255, plunge −2.5, per-tool bit-change pauses. → Done
  8. Abort (any time). "Abort" → Plan = [Halt]: Session → Faulted, the wire halts, and the emulator's motion queue clears — the head stops.

BEFORE three machines, by hand

  1. Click Confirm → job: Drilling.
  2. …set screen: Drill, progress: 0%.
  3. …issue Stream(drill) — refused Busy (dry-run still streaming).
  4. UI shows Drill / 0%; nothing moves to the drill program.
  5. Abort → Halt faults the controller; queued motion keeps running.

AFTER one coordinated transition

  1. Click Confirm → Action: ConfirmRegistration.
  2. transition: Rehearsing → Drilling, Plan = [CancelStream, Stream(drill)].
  3. app.update runs the plan in one effect — clean.
  4. screen(session) derives Drill — and it's true.
  5. Abort → Plan = [Halt]: session Faulted, wire halts, queue clears; test asserts the head stopped.

Everything we did by hand — re-export with the right origin, no-mirror, fiducial G92, post-process for M3 S255, dry-run first, and now sequence the confirm + abort across two machines — becomes a single guided flow where the wrong order is unrepresentable.

11Glossary

Domain vocabulary

Session new
The coordinator value the Lustre model holds: it nests the Job (stage) and PrinterConnection (wire), projects the UI screen, and on a flow Action returns #(Session, Plan) or a typed Rejected. See CONTEXT.
Plan new
The ordered List(printer.Command) a Session transition returns; the app executes it in one effect (never effect.batch). See CONTEXT.
BoardModel
Immutable parse of the KiCad outputs: holes, outline, fiducials, tools — all in board coordinates.
Correspondence
A captured pair {board_point, machine_point} — a board feature and where the printer actually is when the operator locates it (carries machine Z).
PendingAlignment
The in-progress type holding <3 correspondences. Has no transform field — cannot be used to drill.
Alignment
A solved affine Transform2D + a Z surface plane + residuals. Constructible only via alignment.fit from ≥3 non-collinear points.
Residuals
Per-point fit error (rms + max). The trust signal; gates the real run.
PrinterConnection
The single stateful identity: the serial control state machine (pure printer.gleam core + effectful controller.gleam shell over a Backend seam) owning the Web Serial port, the Marlin handshake, live position, and mode.
MarlinEmulator new
The faithful, pure Marlin protocol core behind the Backend seam — line/checksum validation, motor state, M0 block, M114, plus a motion queue, a tick, and an injected envelope. Drives both deterministic e2e tests and a live in-app virtual machine.
GcodeProgram
Generated Marlin G-code for one mode (DryRun | Drill), built from an Alignment.
Job
The session stage machine enforcing the legal order: parse → register → align → dry-run → drill. Embedded by the Session.
